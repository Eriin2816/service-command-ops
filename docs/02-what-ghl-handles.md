# What GHL Handles — Do Not Rebuild

## GHL Is Responsible For:
- Contact records (source of truth for customer data)
- Lead capture (forms, landing pages, ads)
- Conversations (SMS, email, chat widget)
- Sales pipeline and opportunity management
- Appointment booking and calendar
- Missed-call text-back
- Marketing automations
- Review request workflows
- Email/SMS campaigns
- Pipeline stage management

## ServiceOps Integration Points With GHL:
- Receive webhook when opportunity/appointment is job-ready
- Read contact data by GHL contact ID
- Update opportunity status when job is completed
- Trigger GHL automation when estimate is needed
- Trigger GHL automation when review request is appropriate

## What ServiceOps Must NEVER Rebuild:
- A second CRM or contact database
- A second SMS/email system
- A second booking/calendar system
- A second pipeline management tool
- A marketing automation engine
