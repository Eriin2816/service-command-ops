# UI Standards

- Tailwind CSS only — no custom CSS unless absolutely necessary
- shadcn/ui components where available
- Mobile-first for technician view
- Desktop/tablet for admin dashboard
- Consistent navigation from src/components/layout/
- Large tap targets on mobile (min 44px)
- Status badges use enums from src/types/
- Loading states on all async data
- Error states on all async data
- Empty states on all list views
