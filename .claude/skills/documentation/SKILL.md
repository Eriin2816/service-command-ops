# Skill: Documentation

## When to Use
- After any major decision
- After completing a feature
- When memory files are out of date
- When client needs a plain-language explanation

## Steps Claude Should Follow

1. Identify what changed (architecture, feature, decision)
2. Update memory/product-decisions.md if a decision was made
3. Update MEMORY.md summary if product state changed
4. Update relevant docs/ file
5. Update specs/ with implementation notes
6. Ensure README.md is accurate
7. Write client-facing content in plain language

## Output Checklist
- [ ] MEMORY.md updated
- [ ] Relevant memory/ file updated
- [ ] docs/ file updated if applicable
- [ ] specs/ updated if applicable
- [ ] Plain language for non-technical sections

## Mistakes to Avoid
- Technical jargon in client-facing docs
- Leaving MEMORY.md out of date after decisions
- Forgetting to update specs after implementation
